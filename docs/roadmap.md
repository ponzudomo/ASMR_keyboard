# 開発ロードマップ (Roadmap)

ASMR Keyboard の完成に向けたステップバイステップの TODO リストです。

## 🟦 Phase 1: 視差効果のプロトタイプ (Current Priority)
まずは「無地」の状態でも、デバイスの傾きに応じて要素が動く「立体感」を実現します。

- [ ] センサー情報の取得ロジック実装 (加速度/ジャイロ)
- [ ] センサー値の平滑化 (ローパスフィルタ) の導入
- [ ] `Modifier.graphicsLayer` を用いた簡易的な視差移動の実装
- [ ] プレースホルダー（四角形など）での奥行きテスト
- [ ] 視差効果の ON/OFF 切り替え機能（デバッグ用）

## 🟧 Phase 2: キーボード機能の完備 (IME Core)
実用的なキーボードとしての機能を磨き上げます。

- [ ] Shift キーの挙動安定化 (Double tap caps lock 等)
- [ ] Backspace の長押し連続削除
- [ ] Enter キーの実装とアクション対応
- [ ] 記号・数字レイアウトの追加

## 🟩 Phase 3: 打鍵音とフィードバック (Audio ASMR)
キーボードとしての「触感」を音で完成させます。

- [ ] `SoundPool` のセットアップ
- [ ] `res/raw` への音源アセット追加（Wood, Ice 等）
- [ ] キー押下時の音再生ロジックの実装
- [ ] 音のランダムピッチ調整（本物らしさの演出）
- [ ] BGM/環境音（焚き火の音など）のループ再生実装

## 🟨 Phase 4: テーマの肉付け (Visual Polish)
`DESIGN.md` に基づいた各テーマ固有の演出を実装します。

- [ ] **Frozen Serenity:** ヒビ割れ演出とパーティクル
- [ ] **Wood:** 深い沈み込みと木目のテクスチャ
- [ ] **Slime Pop:** 粘性のある変形アニメーション
- [ ] **Cozy Ember:** 火の粉と赤熱エフェクト

## 🟪 Phase 5: 最適化と設定 (UX & Settings)
ユーザーが自分好みに調整できるようにします。

- [ ] 設定画面の作成 (テーマ選択、音量、視差の強さ)
- [ ] バッテリー消費の最適化 (センサー・アニメーションの休止)
- [ ] 9-patch 等を用いたアセットの軽量化
