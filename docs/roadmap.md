# 開発ロードマップ (Roadmap)

ASMR Keyboard の完成に向けたステップバイステップの TODO リストです。

## ✅ Phase 1: 視差効果と物理挙動の完成 (Completed)
3DSのような立体感と、物理的に正しい沈み込み挙動を実現しました。

- [x] 6レイヤー構造の導入 (L1-L6)
- [x] 絶対的なデバイス傾きに基づいた差動パララックスの実装
- [x] プレス時の「穴に吸い込まれる」沈み込み（Sinking）の実装
- [x] センサー値の平滑化 (ローパスフィルタ) と基準角補正の導入
- [x] 最大移動量の制限 (Clamping) による視覚的安定性の確保

## ✅ Phase 2: キーボード基本機能の完備 (Completed)
実用的なキーボードとしてのコア機能を実装しました。

- [x] Backspace の長押し連続削除
- [x] Enter キーの実装
- [x] 数字/記号モードの切り替え機能と基本レイアウト

## 🟩 Phase 3: 打鍵音とフィードバック (Audio ASMR) (Current Priority)
キーボードとしての「魂」である音を実装し、触感を完成させます。

- [ ] **SoundPool 管理システムの構築**
    - [ ] 音声再生用のシングルトン/管理クラスの作成
    - [ ] キーの種類に応じた音の出し分けロジック
- [ ] **打鍵音のバリエーション実装**
    - [ ] 打鍵ごとのわずかなピッチ変更 (0.95x ~ 1.05x)
- [ ] **仮のシステム音での動作確認**
- [ ] `res/raw` への本番用音源アセット追加（Wood, Ice 等）
- [ ] BGM/環境音（焚き火の音など）のループ再生実装

## 🟨 Phase 4: テーマの肉付け (Visual Polish)
各テーマに固有の豪華な視覚エフェクトを追加します。

- [ ] **Frozen Serenity:** ヒビ割れ演出とパーティクル
- [ ] **Slime Pop:** 粘性のある変形アニメーション（リファクタリング）
- [ ] **Cozy Ember:** 火の粉と赤熱エフェクト

## 🟪 Phase 5: 最適化と設定 (UX & Settings)
- [ ] 設定画面の作成 (テーマ選択、音量、視差の強さ)
- [ ] バッテリー消費の最適化 (センサー・アニメーションの休止)
- [ ] Shiftキーの高度な挙動（Caps Lock等）の安定化
