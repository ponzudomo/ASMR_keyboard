# アセット管理 (Asset Management)

このドキュメントは、プロジェクトで使用する音源、画像、およびその他のリソースに関する仕様を定義します。

## 1. 命名規則 (Naming Convention)

すべてのリソースファイルは、以下の規則に従って命名します。これにより、どのテーマのどの要素であるかが一目でわかります。

`{type}_{theme}_{variant}_{number}.{extension}`

- **type:** `sound`, `texture`, `icon` など
- **theme:** `wood`, `ice`, `slime` など
- **variant (任意):** `tap`, `release`, `background` など
- **number (任意):** `01`, `02` など（複数パターンある場合）

**例:**
- `sound_wood_tap_01.ogg` (Woodテーマのタップ音1)
- `texture_ice_background.jpg` (Iceテーマの背景画像)

---

## 2. 音源 (Audio Assets)

### フォーマット
- **形式:** OGG (`.ogg`) を推奨。
- **品質:** 44.1kHz, 16-bit, Mono。ステレオである必要はありません。
- **ファイルサイズ:** 1ファイルあたり最大でも 100KB 程度に抑える。

### 仕様
- **低遅延:** `SoundPool` での再生に最適化するため、ファイル冒頭の無音部分は可能な限り削除します。
- **バリエーション:** 同じ操作（例：タップ）でも、3〜5種類の微妙に異なる音を用意することで、機械的な繰り返し感をなくします。
- **BGM (環境音):** ループ再生することを想定し、始点と終点が自然につながるように編集します。（例: CozyEmberの薪のはぜる音）

---

## 3. 画像 (Image Assets)

### フォーマット
- **背景:** WebP (`.webp`) または JPG (`.jpg`)。
- **アイコン・エフェクト:** WebP (`.webp`) または PNG (`.png`)。透過をサポート。

### 解像度
- **背景テクスチャ:** 汎用性を考慮し、`2048x2048` ピクセル以上を推奨。
- **アイコン:** `192x192` (xxhdpi) を基準に作成。

---

## 4. 格納場所

- **音源:** `app/src/main/res/raw/`
- **画像:** `app/src/main/res/drawable/`
